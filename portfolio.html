Multi Chat Server - University Assignment

  • Date Started and Ended: February 2010
  • Contribution: Sole Developer
  • Technologies Used: Java SE 6 (including Swing and sockets)
  • Description: This client/server application was a class assignment in which I had to create a chat server application along with its complementary chat client application.  The server is responsible for managing client connections and relaying chat messages to all clients.  The server's application window maintains a log of all activities.  The client requests a unique chat name (for identification on the server) and the server's address.  Once it connects to the server the client can send and receive messages from other connected clients (via the server).
  • Code Samples: Multi Chat Server on Github
  • Screenshots:
  • Screenshot Comments:
    • This screenshot simply shows the operation of the chat server along with two connected clients.
    • All of these windows are updated dynamically with the server log acting as a reference to how and where messages are being sent.

AuthAnvil Soft Token for iPhone - University Project Collaboration with Scorpion Software Corp.

  • Date Started and Ended: January - April 2009
  • Contribution: Partner in a Two (2) Person Team, Developer, GUI Design and Implementation
  • Technologies Used: Objective-C (iPhone OS 2.1), ANSI C, SVN
  • Description: This mobile application was a term project in collaboration between the University of the Fraser Valley and Scorpion Software Corp.  It allows an iOS device to become a one time password generator (software token, “soft token” for short) for accessing sensitive systems protected with AuthAnvil Two Factor Authentication (2FA).  See www.scorpionsoft.com for more information.
